Technology

As you notice, the site and documentation is still under construction. However, here are some specs:

  • User login : 60K rounds of PBKDF2-HMAC-SHA512.
  • Secret hash: Hash of answers is encrypted with a derivative of password hash + PIN using 10K rounds of PBKDF2-HMAC-SHA512 + AES-256 and kept in DB.
  • Encrypted PIN: Encrypted with user information hash using 10K rounds of PBKDF2-HMAC-SHA512 + AES-256 and kept in DB.
    • Master Private Key: Generated with PIN + User information hash through 30K rounds of PBKDF2.
  • Wallet Addresses: Multi-signature key pairs generated through cross-matching children of the master node. Receiving addreses are generated and managed in the server by using the master public key.

All operations takes place in client-side JS and all password derivation is completed with user's unique salt.

This scheme makes password and PIN recovery possible by providing high entropy for all encrypted data.
